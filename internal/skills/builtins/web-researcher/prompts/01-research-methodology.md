# Web Research Methodology

You have the web-researcher skill. Follow this workflow for all browser-based research.

## Research Workflow

1. **Define search terms** — Before searching, identify 3-5 specific search queries. Use precise terms over broad ones. Consider synonyms and alternative phrasings.
2. **Search broadly** — Run multiple searches to gather a diverse set of sources. Do not rely on a single query or the first few results.
3. **Evaluate credibility** — Assess each source before extracting information (see 02-source-evaluation.md).
4. **Extract facts** — Record specific facts, data points, and claims from each source. Note the source URL and access date for each.
5. **Cross-reference** — Verify key claims against at least two independent sources. Note when claims are single-sourced.
6. **Synthesize** — Combine findings into a coherent summary. Distinguish well-supported conclusions from tentative ones.
7. **Write output** — Save research to `output/` with a full source list at the end.

## Rules

- **Never present single-source claims as established facts.** If only one source supports a claim, state it as "according to [source]" rather than as fact.
- **Record URLs** — Every cited source includes its URL and the date accessed.
- **Note contradictions** — When sources disagree, present both positions and note the disagreement rather than choosing one.
- **Time-bound claims** — Note when information is time-sensitive. Prices, statistics, and policies change.
- **Stay on topic** — Web research can lead to tangential discoveries. Note tangents in memory for later but stay focused on the original question.
