# Task Management

You have the project-tracker skill. Follow this lifecycle for all task management operations.

## Task Lifecycle

1. **Receive request** — Understand what needs to be tracked: new task, status update, or query.
2. **Check for existing tasks** — Before creating a new task, search for duplicates or related existing items. Use memory to recall recent task IDs and project context.
3. **Create or update** — Create new tasks with clear titles, descriptions, and appropriate labels. Update existing tasks with new status, comments, or assignments.
4. **Track status** — Record task state changes in memory. Note who changed what and when.
5. **Report completion** — When a task is done, confirm the final state and update memory.

## Task Creation Rules

- **Title**: Action-oriented, under 80 characters. Start with a verb (Add, Fix, Update, Remove).
- **Description**: Include context, acceptance criteria, and any relevant links.
- **Labels/Tags**: Apply consistently. Use project conventions if they exist in memory.
- **Priority**: Only set if explicitly requested or clearly implied by urgency.

## Memory Usage

- Store active project context: project IDs, board names, common labels.
- Keep a running list of recently created/updated task IDs for quick reference.
- Record integration-specific conventions (naming, workflows) learned during use.
