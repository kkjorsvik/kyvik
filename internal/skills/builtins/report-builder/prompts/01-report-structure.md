# Report Structure

You have the report-builder skill. Follow this template for all reports.

## Standard Report Template

Every report should include these sections in order:

1. **Title and Date** — Report name and generation date. Use format: `YYYY-MM-DD - Report Title`.
2. **Executive Summary** — 3-5 sentences covering the most important findings and any recommended actions. A reader should understand the report's conclusions from this section alone.
3. **Methodology** — How the data was gathered, what sources were used, and what time period is covered. Include any limitations.
4. **Findings** — Present the data and observations. Use tables, lists, and clear headings. Organize by theme or chronologically, whichever fits the content better.
5. **Analysis** — Interpretation of the findings. What do the patterns mean? What are the implications?
6. **Recommendations** — Specific, actionable next steps based on the analysis. Prioritize by impact.
7. **Appendices** — Raw data, detailed tables, or supplementary information that supports the main report but is too detailed for the body.

## File Naming

Write reports to `output/` using this naming convention:
- `output/YYYY-MM-DD-report-title.md` for single reports
- `output/YYYY-MM-DD-report-title/` directory for reports with multiple files

## Formatting

- Use Markdown for all reports.
- Use tables for comparative data.
- Use headings consistently (## for sections, ### for subsections).
- Keep paragraphs short (3-5 sentences).
